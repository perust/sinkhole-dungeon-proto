# Unlit Halls — Implementation Plan v0.2

> 기준 문서: `GAME_CONCEPT_V0.2.md`  
> 목표: 현재 프로토타입을 `회수물 줍기 → 귀환 → 판매 → 강화` 중심에서 **괴물을 피해 더 깊이 가기 위한 생존 탐험 루프**로 전환한다.

---

## 0. v0.2 성공 기준

v0.2는 많은 콘텐츠를 넣는 패스가 아니다. 핵심 감각 하나를 검증한다.

> **괴물을 만났을 때 플레이어가 “계획이 망했다. 뭘 포기하지?”라고 느끼는가?**

이 감각이 생기면 v0.2 성공이다.

### 플레이어가 느껴야 하는 것

- 괴물은 싸워서 이기는 대상이 아니다.
- 괴물은 경로와 계획을 망가뜨린다.
- 물건을 들고 있을수록 더 위험하다.
- 어떤 물건은 팔기보다 분해하거나 버리는 게 낫다.
- 실패해도 즉시 끝나지는 않지만, 시간/장비/아이템 손실이 남는다.
- 지상에서는 다음 잠수를 준비해야 한다.

---

## 1. 전체 작업 순서

### Phase 1 — Monster Threat Pass

괴물을 단순 이벤트가 아니라 맵 위 위협으로 만든다.

### Phase 2 — Survival Item Pass

회수물을 판매용이 아니라 생존/정보/미끼 자원으로 확장한다.

### Phase 3 — Minimal Surface Hub Pass

지상 화면을 `내려가기 / 장비 / 소문` 중심의 준비 허브로 재정리한다.

### Phase 4 — Gear Damage & Recovery Pass

괴물 실패와 루프 이후 손상이 장비 내구도/수리로 이어지게 한다.

### Phase 5 — Three-Day Loop Scaffold

완전한 3일 루프 구현 전, Day/time 상태와 첫 재앙 이벤트를 얹을 수 있는 구조를 만든다.

## 2. Phase 1 — Monster Threat Pass

### 2-1. 목표

현재 괴물 조우는 선택지만 누르면 큰 피해 없이 넘어갈 수 있다. v0.2에서는 괴물이 플레이어의 동선과 아이템 판단을 압박해야 한다.

### 2-2. 구현 항목

#### 1) 맵 위 괴물 상태 추가

`run`에 최소 상태를 추가한다.

```js
run.monsters = [
  {
    id: 'shadow-1',
    nodeId: 3,
    state: 'roaming', // roaming | alerted | feeding | stunned
    targetNodeId: null,
    cooldown: 0,
  }
];
```

초기 v0.2에서는 1층/초반 구간에 괴물 1마리만 둔다.

#### 2) 괴물 이동 틱

- 플레이어가 이동할 때 괴물도 1스텝 또는 확률적으로 이동한다.
- 플레이어가 소리를 내거나 위험물을 들고 있으면 괴물이 플레이어 쪽으로 이동할 확률이 오른다.
- 텍스트 읽기 게이트 중에는 이동하지 않는다.

#### 3) 근처 감지

- 괴물이 현재 노드와 인접 노드에 있으면 미니맵에 흔적 표시.
- 직접 이름은 표시하지 않는다.
- 표현은 `기척`, `젖은 발소리`, `검은 흔적` 정도로 시작한다.

#### 4) 시야 실루엣

- 같은 노드 또는 전방/연결 노드에 괴물이 있으면 stage에 검은 실루엣 상태를 표시한다.
- 기존 회수자 스프라이트를 직접 노출하기보다 어두운 실루엣/그림자 레이어로 처리한다.

#### 5) 조우 상태

괴물과 같은 노드에 들어가거나 괴물이 플레이어 노드로 들어오면 `monsterEncounter` 상태로 전환한다.

```js
run.monsterEncounter = {
  monsterId: 'shadow-1',
  reason: 'same-node',
  canRetreat: true,
  canDrop: run.bag.length > 0,
  canHide: true,
  canDash: true,
};
```

#### 6) 조우 선택지

상황에 따라 2–4개만 노출한다.

- `뒤로 도망친다`
- `물건을 버리고 달아난다`
- `숨는다`
- `뛰어 지나친다`
- `장비를 쓴다`는 장비가 있을 때만

#### 7) 무피해 통과 제거

모든 선택은 비용이 있어야 한다.

- 후퇴: 시간 + 조명 손실, 이전 노드로 이동
- 버리기: 아이템 손실, 괴물 feeding 상태
- 숨기: 멘탈 손실, 실패 시 더 큰 위험
- 돌파: 성공률 낮음, 실패 시 기절
- 장비 사용: 소모품/내구도 손실

### 2-3. 판정 기본안

초기값은 과감하게 단순하게 둔다.

```js
const ENCOUNTER_ACTIONS = {
  retreat: { base: 0.65, lightCost: 8, mentalCost: 4, timeCost: 12 },
  drop:    { base: 0.90, lightCost: 2, mentalCost: 2, timeCost: 6  },
  hide:    { base: 0.55, lightCost: 0, mentalCost: 12, timeCost: 18 },
  dash:    { base: 0.25, lightCost: 10, mentalCost: 8, timeCost: 8  },
};
```

보정 요소:

- 가방이 무거우면 `retreat`, `dash` 하락
- 회수물이 많으면 괴물 추격 확률 상승
- 조명이 낮으면 `hide` 일부 상승, `dash` 하락
- 멘탈이 낮으면 전체 하락
- 장비/소문으로 괴물 규칙을 알면 특정 선택 상승

### 2-4. 실패 결과

조우 판정 실패 시 바로 게임오버가 아니라 기절/구조 이벤트로 보낸다.

- 회수물 대부분 손실
- 장비 내구도 손상
- 시간이 흐름
- 같은 층 어두운 구석에서 깨어날 수 있음
- 일부 확률로 지상 구조/나쁜 조사자/위원회 구조반 이벤트

### 2-5. 검증 시나리오

- 괴물이 인접 노드에 있을 때 미니맵 흔적이 표시되는가?
- 괴물과 같은 노드가 되면 조우 선택지가 뜨는가?
- `물건을 버리고 달아난다` 선택 시 실제 아이템이 사라지는가?
- `뒤로 도망친다` 선택 시 이전 노드로 이동하고 시간/조명이 줄어드는가?
- 실패 시 기절/구조 이벤트로 이어지는가?
- 조우 후 아무 손실 없이 내려가는 경로가 사라졌는가?

## 3. Phase 2 — Survival Item Pass

### 3-1. 목표

회수물을 단순 판매물이 아니라 더 깊이 가기 위한 판단 자원으로 만든다.

### 3-2. 아이템 속성 추가

기존 `ITEM_TABLE`에 v0.2 속성을 추가한다.

```js
{
  name: '비상 배터리',
  slots: 1,
  value: 4,
  tier: 'common',
  icon: 0,
  role: 'salvage', // sell | salvage | info | dangerous
  lightGain: 18,
  repairGain: 0,
  noise: 2,
  lure: 1,
  infoKey: null,
}
```

### 3-3. 초반 아이템 6종

1. **낡은 계측기**
   - role: `sell`
   - 안전한 판매용
   - 초반 RP 확보 담당

2. **비상 배터리**
   - role: `salvage`
   - 현장 분해 시 조명 회복
   - 판매 가치는 낮음

3. **금속 공구함**
   - role: `salvage`
   - 임시 수리/미끼
   - 무겁고 도망에 불리

4. **찢어진 조사 기록**
   - role: `info`
   - 소문/분석에서 다음 층 힌트
   - 판매 가치는 거의 없음

5. **구조자의 표식 조각**
   - role: `info`
   - 구조자 단서
   - 특정 NPC 대화 해금 후보

6. **검게 식은 유리병**
   - role: `dangerous`
   - 들고 있으면 괴물 반응 상승
   - 분석하면 연구시설 단서
   - 버리면 괴물을 오래 붙잡음

### 3-4. 현장 분해 액션

던전에서 특정 아이템을 선택하면 `분해한다` 액션을 제공한다.

초기 범위:

- `비상 배터리` → 조명 회복
- `금속 공구함` → 장비 내구도 일부 회복 또는 소음 미끼 제작
- 위험물은 현장 분해 불가, 분석/버리기만

### 3-5. 버리기 액션

괴물 조우 중 `물건을 버리고 달아난다`는 단순히 가장 비싼 물건 자동 손실이 아니라, v0.2에서는 가능하면 선택형으로 바꾼다.

1차 구현은 단순화:

- 위험물 우선
- 그다음 lure 값이 높은 물건
- 그다음 value 높은 물건

후속 구현에서 직접 선택 UI로 확장한다.

### 3-6. 검증 시나리오

- 배터리를 판매하지 않고 분해할 수 있는가?
- 분해 후 조명이 실제로 회복되는가?
- 위험물을 들고 있으면 괴물 반응/추격이 강해지는가?
- 괴물 조우 중 버린 물건이 실제로 사라지는가?
- 정보용 아이템은 팔기보다 보관하고 싶게 느껴지는가?

## 4. Phase 3 — Minimal Surface Hub Pass

### 4-1. 목표

지상을 단순 판매/강화 화면이 아니라 다음 잠수를 준비하는 허브로 바꾼다.

### 4-2. 첫 허브 메뉴

처음에는 복잡한 마을 지도 대신 3개 메뉴만 둔다.

1. **내려가기**
   - 현재 장비 상태
   - 현재 도달 가능 층
   - 위원회 경고/허가 문구

2. **장비**
   - 조명 충전
   - 장비 수리
   - 기본 소모품 준비

3. **소문**
   - 구조자 단서
   - 최하층 괴담
   - 괴물 회피 힌트
   - 첫 재앙 NPC 후보 대화

### 4-3. 기존 화면과의 연결

기존 시작 화면을 허브화한다.

- 현재 `입장` 버튼은 `내려가기` 메뉴로 유지
- `RP/강화/의심도/진실 조각`은 축소하거나 정보 패널로 이동
- 판매/강화 화면은 일단 유지하되, 장비 메뉴로 흡수할 수 있게 구조를 분리

### 4-4. 소문 데이터

초기 소문 6개 정도만 둔다.

- 최하층 소문 2개
- 구조자 관련 소문 2개
- 괴물 회피 힌트 1개
- 위원회 불신/통제 소문 1개

### 4-5. 검증 시나리오

- 시작 화면에서 내려가기/장비/소문 흐름이 이해되는가?
- 소문을 읽는 동안 시간/위험이 흐르지 않는가?
- 장비 화면에서 수리/충전의 필요성이 보이는가?
- 허브가 복잡한 메뉴가 아니라 다음 잠수 준비처럼 느껴지는가?

## 5. Phase 4 — Gear Damage & Recovery Pass

### 5-1. 목표

실패가 단순 보상 손실이 아니라 장비 상태 악화로 남게 한다.

### 5-2. 장비 내구도

초기에는 모든 장비를 세분화하지 말고 2개만 둔다.

```js
meta.gear = {
  lightDurability: 100,
  kitDurability: 100,
};
```

- `lightDurability`: 조명 최대치/회복 효율에 영향
- `kitDurability`: 괴물 조우/수리/분해 효율에 영향

### 5-3. 손상 이벤트

- 괴물에게 잡힘: 조명 내구도 -15~30
- 나쁜 조사자 구조: kit 내구도 -20~40
- 루프 재시작: 들고 있던 장비 내구도 일부 손상
- 무리한 돌파 실패: 둘 다 손상 가능

### 5-4. 수리

지상 장비 메뉴에서 RP 또는 부품을 사용해 수리한다.

초기 단순안:

- RP로 수리
- 금속 공구함 분해 시 일부 수리

### 5-5. 검증 시나리오

- 괴물 실패 후 장비 내구도가 실제로 내려가는가?
- 내구도가 낮으면 다음 잠수가 불리한가?
- 지상에서 수리할 수 있는가?
- 수리비 때문에 아이템 판매/분해 판단이 생기는가?

## 6. Phase 5 — Three-Day Loop Scaffold

### 6-1. 목표

완전한 루프 엔딩을 만들기 전, 시간 상태와 첫 재앙 이벤트를 넣을 수 있는 구조를 만든다.

### 6-2. 시간 상태

```js
meta.loop = {
  cycle: 1,
  day: 1,
  minute: 8 * 60, // 08:00
  firstDisasterSeen: false,
};
```

### 6-3. 시간 흐름 규칙

- 방 이동: +5~15분
- 층 이동: +20~40분
- 휴식: +1~3시간
- 기절: +3~12시간
- 지상 장비/소문 메뉴: 실제 시간처럼 조금씩 흐르되, 읽기 게이트 중에는 정지

### 6-4. 읽기 중 시간 정지

기존 `dialogue` 게이트가 active일 때는 시간 진행/괴물 이동을 멈춘다.

### 6-5. 첫 재앙 이벤트 자리

Day 3 밤 또는 일정 시간 이후:

- 직접 구현은 최소화
- `firstDisasterSeen === false`이면 재앙 프롤로그 화면으로 이동
- 이후 Day 1로 복귀
- 장비 일부 손상
- 특정 NPC 소문 상태 리셋/변형

### 6-6. 검증 시나리오

- 이동/층 이동/기절에 따라 시간이 증가하는가?
- 대화/상황 텍스트 읽기 중에는 시간이 멈추는가?
- Day 3 밤 조건을 감지할 수 있는가?
- 재앙 이벤트 후 Day 1로 돌아오는 구조가 가능한가?

## 11. 완료 정의

v0.2 Systems Pass는 다음 조건을 만족하면 완료로 본다.

- 괴물을 피하기 위해 실제로 경로를 바꾸거나 아이템을 버리게 된다.
- 아이템을 팔지 않고 분해/보관/버리는 선택이 의미 있다.
- 실패가 단순 게임오버가 아니라 시간/장비/아이템 손실로 남는다.
- 지상에서 다음 잠수를 준비하는 이유가 생긴다.
- 3일 루프를 얹을 수 있는 시간 상태가 코드에 존재한다.
- 모바일 화면에서 조우/허브/던전 UI가 잘리지 않는다.
- `node --check`, `git diff --check`, 로컬 브라우저 스모크가 통과한다.
