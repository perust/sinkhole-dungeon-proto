# Phaser 1층 슬라이스 (실험)

도심 싱크홀 회수 프로토타입의 **1층 감각**만 떼어내 Phaser 3로 만든 작은 격리 실험입니다.
루트의 DOM 프로토타입(`index.html`, `styles.css`, `src/game.js`)과 **완전히 분리**되어 있으며,
그 파일들을 전혀 건드리지 않습니다. 이 폴더만으로 독립 실행됩니다.

## 무엇을 보여주나

- **어두운 오버헤드 탐색 뷰포트** — 캔버스에 한 줌의 방(노드)과 복도(간선)를 그리고, 밝혀진 곳만 보인다.
- **이동** — 방향키/`WASD`, 또는 인접한 방을 클릭. 각 이동은 시간을 쓴다.
- **숨은 어두운 형체의 기척** — 정확한 위치·수치·스프라이트는 절대 뜨지 않는다.
  그래프 거리를 `잠잠하다 / 먼 어둠에서 발소리가 스친다 / 어두운 형체가 가까이 있다 / 바로 곁이다` 네 단계로만
  요약하고, 가장자리 어둠과 짧은 붉은 맥동으로 압박을 전한다.
- **맨손 1칸 회수** — 손은 한 칸뿐이라 회수물 하나만 쥔다.
- **귀환/지상 결과 경로** — 입구로 돌아와 `귀환`하면 결과 화면이 뜬다. 손에 쥔 것을 도로 빼앗기면 실패.

## 규칙 한눈에

- `낮은 창고`에 회수물이 하나 놓여 있다. `E`(또는 `줍기`)로 손에 쥔다.
- 회수물을 쥐는 순간부터 어두운 형체가 그 물건을 좇아 다가온다(그전엔 배회).
- 어두운 형체와 같은 칸에서 마주치면 손에 쥔 것을 도로 가져간다 → 맨손으로 조사 실패.
- 조명은 이동마다 줄어 뷰포트가 어두워진다(압박 자원). 실패의 직접 원인은 아니다.
- `입구`에서 `Enter`(또는 `귀환`)로 지상에 오른다. 빈손으로도 오를 수 있다.

## 로컬에서 실행

정적 파일뿐이라 서버가 꼭 필요하진 않습니다. 두 방법 중 편한 쪽으로:

1. **바로 열기** — `phaser-slice/index.html`을 브라우저로 연다(더블클릭). Phaser는 CDN에서 받습니다.
2. **간단한 정적 서버**(권장, 캐시/보안 정책이 깔끔) — 이 폴더에서:

   ```bash
   cd phaser-slice
   python3 -m http.server 8000
   # 브라우저에서 http://localhost:8000 접속
   ```

   Node가 있다면 `npx serve .` 등 아무 정적 서버나 됩니다.

인터넷 연결이 있어야 CDN Phaser가 로드됩니다(오프라인이면 캔버스가 비어 있습니다).

## 파일 구성

```
phaser-slice/
├─ index.html      앱 셸: 상태바 · 캔버스 마운트 · 조작 독 · 오버레이
├─ styles.css      이 슬라이스 전용 스타일(루트와 무관)
├─ src/
│  ├─ floor.js     순수 데이터 + 그래프 헬퍼(노드/간선/거리/기척 · Phaser 비의존)
│  └─ main.js      Phaser 씬 + 렌더 + 입력 + HTML 배선
└─ README.md
```

## 의도적으로 옮기지 않은 것

작게 유지하려고 본 게임의 아래 시스템은 이 슬라이스에서 뺐습니다(설계상 제외):

- 판매처 선택(위원회/암시장/가족 반환), RP·의심도·단서·오늘의 의뢰
- 가방 크기 구매, 조명/장비 손질, 저장/이어하기
- 층 하강(2층+), 방 타입 다수(감시소·실종자 흔적·무너진 통로 상호작용), 출구 선택
- 생존자·변이·끌개(버린 물건 자국) 같은 확장 루프
- 회수물 태그 밸런스(heat/noise/fragile/marked/truth)와 엔딩

이 슬라이스는 **1층의 첫 감각**(어둠 속 이동 · 보이지 않는 압박 · 맨손 한 칸 · 물건을 쥐고 귀환)만
플레이 가능한 형태로 확인하기 위한 것입니다. 규칙·상수·저장 형식·프로덕션 배포 경로는 건드리지 않습니다.
