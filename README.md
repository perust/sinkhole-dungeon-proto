# 도심 싱크홀 — 회수 잠수 프로토타입

가볍고 직관적인 던전 회수(extraction) 게임의 첫 플레이어블 프로토타입입니다.

## 핵심 훅

> 던전이 내가 훔친 물건을 다시 가져가려 한다.

플레이어는 도심 싱크홀 아래 던전에 들어가 회수물을 훔치고, 어두운 형체가 물건을 되찾으러 쫓아오기 전에 지상으로 귀환해 판매합니다. 얻은 RP로 정해진 크기의 가방을 사거나 조명/장비를 손보고 다시 더 깊이 내려갑니다.

## 구현된 루프

1. `입장`으로 던전 1층 진입 — 층마다 5~7개 노드짜리 작은 맵이 생성된다
2. 갈림길 선택: 현재 노드의 출구로 동적 구성(예: `왼쪽 문` / `정면 복도` / `환풍구` / `계단 아래로` / 막다른 `관리실`)
3. 도착한 노드에 회수물이 있으면 `줍기`로 가방에 추가
4. 내부 위험도 상승. 화면에는 숫자 대신 `기척`만 남음
5. 몬스터 이벤트 대응: 복도 끝의 소리 / 갈림길을 스치는 발소리 / 움직이면 들킴(`기다리기`로 완화)
6. `계단 아래로` / `버리고 도망` / `들고 귀환` 판단
7. 귀환 성공 시 판매처 선택: `위원회` 또는 `암시장`
8. 판매 결과로 RP, 의심도, 단서, 의뢰 결과가 변화
9. 가방 구매 또는 조명/장비 손질
10. `다시 내려가기`로 다음 조사 시작
11. 내부 위험도가 한계에 닿으면 어두운 형체가 가방을 되찾아가고 조사 실패, 일부 보상만 유지

## v2 고도화 반영

- 판매처 선택: 위원회는 안전/저가/의심도 감소, 암시장은 고가/의심도 상승/단서 해금
- 지상 의심도: 장기 압박 자원
- 단서: 회수물별 미스터리 해금
- 오늘의 의뢰: 반복 진입의 단기 목표
- 다음 목표/거리 뉴스/조사 요약: 플레이어가 왜 다시 들어가야 하는지 명확화
- 저장/이어하기: RP, 강화, 최고 깊이, 의심도, 단서, 오늘의 의뢰 진행 자동 저장
- 모바일 앱형 UI: PWA manifest, 앱 셸, 상단 상태바, 하단 조작 독
- 1인칭 던전 탐색: 설명 패널을 줄이고, 갈림길 선택 → 전진 → 발견 흐름으로 변경
- 작은 맵 생성: 층마다 5~7개 노드(출구/방 유형/아이템/몬스터)로 구성된 미로
- 동적 갈림길: 항상 3개 고정이 아니라 현재 노드 출구에 따라 2~4개의 상황형 선택지
- 몬스터 시야 이벤트 3종: 직선 끝에 보임 / 갈림길을 지나감 / 움직이면 들킴 + `기다리기`
- 시야창 확대 + 줍기 전/후 애니메이션 + `앞으로` 전진 시 문 접근 연출 강화
- 비가시 어두운 형체 연출: 어두운 형체 스프라이트/추적도 숫자/위험 바를 숨기고 `기척` 감각 힌트만 표시
- 장비 화면 고정 UI: 판매/의뢰/뉴스가 길어져도 장비 3종과 `다시 내려가기` CTA가 화면 안에 고정
- 공간 배치형 장소 선택: 장소 버튼을 좌/중/우 방향 패드처럼 배치하고, 긴 상황 설명은 가방 아래 cue 박스로 분리
- 아이템 태그 v1: 물건별 `heat`(암시장 의심도)·`noise`(집기 소음 low/medium/high)·`fragile`(버리면 파손). 인던에서는 수치 대신 `소리가 클/깨질 것 같다` 힌트만 노출
- 단서 6개 엔딩 v1: 마지막 단서를 처음 해금하면 장비 화면 대신 엔딩(리빌 3줄 + 6/6)을 띄우고, `심층 조사 계속`(장비 루프 유지)/`기록 초기화`를 제공. `endingSeen` 저장 플래그로 확인 여부 기록

- 끌개 v1 (버린 물건 자국): `버리고 도망`이 물건을 없애지 않고 바닥에 임시 자국으로 바꾼다. 던전은 미끼가 아니라 되찾을 물건을 좇아 그 칸으로 향하고, 유예 안에 회수해 간다. 플레이어가 놈보다 먼저 그 칸에 닿으면 다시 챙길 수 있다(`그냥 둔다`를 고르면 자국은 그대로 놈의 몫이 된다)
- 방 타입 확장 v1 (무너진 통로 · 위원회 감시소): `무너진 통로`는 고위험/고보상 방으로, `잔해를 넘어간다`(빛·멘탈 소모·소음↑, 방에 물건이 없으면 더 나은 회수물이 드러남)/`낮게 돌아간다`(시간·빛은 들지만 조용)/`포기한다`를 고른다. `위원회 감시소`는 드물게 등장하며 의심도와 상호작용한다 — `기록을 지운다`(의심도 소폭↓·소음)/`단말을 뒤진다`(단서 로그나 물건 기회, 의심도 높으면 위험↑)/`그냥 지나간다`. 인던에서 수치·비밀을 공짜로 노출하지 않고 단서 한 줄만 흘린다
- 출구 선택 v1 (공식 · 균열 · 암시장 통로): 판매처를 고르기 전, 지상으로 나가는 세 갈래 길을 고른다. `공식 출구`는 안전하지만 짐이 뜨겁거나 의심도가 높으면 검문에 걸려 의심도가 조금 오르고, `균열 출구`는 검문을 피해 의심도가 조금 줄지만 파손되기 쉬운 물건이 긁혀 견적이 깎이며, `암시장 통로`는 통행료(-8 RP)를 내는 대신 암시장 판매 꼬리(의심도)를 덜어낸다(빈손이면 통행료 없음). 보정은 무작위 없이 견적·의심도에 그대로 반영
- 왜곡 변이 v1 (균열 시야 · 검은 손): 아주 뜨거운 유물(heat≥15, 안정화 코어·봉인 유물)을 하나라도 들고 지상으로 나오면, 충분히 깊이 내려갔거나(2층 이상) 방을 여럿 밟은(3곳) 조사에 한해 몸에 영구 흔적이 새겨진다(`meta.mutations`, 자동 저장, 조사당 한 번 판정). 지급은 무작위 없이 `균열 시야` → `검은 손` 순으로 미보유 첫 변이를 주고, 가방의 물건을 없애거나 비밀을 공짜로 바꾸지 않는다. 하나가 이득과 대가를 함께 준다 — `균열 시야`는 지도공을 쓰지 않을 때 갈림길 큐에 앞쪽 틈/계단/물건 방향을 아주 짧게 짚어 주지만(`균열 시야:` 한 줄, 낮은 정밀도), 균열 출구에서 길을 잘못 짚어 파손되기 쉬운 짐 긁힘이 조금 커진다(15%→20%). `검은 손`은 휘어진 캐비닛을 더 쉽게 열지만(조명 소모 -3→-1), 공식 출구 검문에 걸리면 손이 눈에 띄어 의심도가 +1 더 오른다. 시작 화면에 `몸에 남은 흔적`으로 표시한다(더 큰 변이 트리·확장은 후속)
- 생존자 시스템 v1 (정비공 · 의무병 · 지도공 · 전 위원회 직원): 던전에서 드물게 갇힌 사람을 만난다 — 휘어진 사물함 뒤에서 뻗는 손, 무너진 선반에 눌린 다리, 휘어진 벽판 뒤에서 젖은 지도를 쥔 손, 부서진 ID 게이트에 낀 팔. `꺼내준다`로 구출하면 빛·멘탈을 쓰고 소음이 나지만, 그 사람이 지상으로 따라와 조사를 넘어 이어진다(`meta.survivors`, 자동 저장). `정비공`은 장비 강화 비용을 25% 깎아 주고(장비 화면에 `정비공 할인` 표기), `의무병`은 기절했을 때 오르는 의심도를 덜고 위로 보상을 조금 더 챙겨주며(실패 화면에 적용 문구 노출), `지도공`은 갈림길에서 각 출구 앞쪽 방의 특징(계단·물건 유무·방 종류)을 방향 기호와 함께 미리 짚어 준다(상황 큐에 `지도공 표시:` 한 줄, 낮은 정밀도). `전 위원회 직원`은 위원회 감시소에서 `봉쇄 코드를 넣는다` 선택지를 열어 소음·위험 없이 의심도를 조금 덜고 봉쇄문/재봉인 단서 한 줄을 흘리며(비밀은 공짜로 주지 않음), 공식 출구 검문에 걸렸을 때 미리 넣어 둔 통행 암구호로 오르는 의심도를 2 덜어낸다(짐이 뜨겁거나 의심도가 높으면 여전히 조금은 오른다). 구출하지 않아도 선택은 기록된다(미구출 후속 v1): `위치만 표시한다`는 벽에 분필 표시를 두고 물러나 소음·의심 없이 시작 화면에 `표시해 둔 위치 N곳`으로만 표시되고, `그냥 지나간다`는 등을 돌린 대가로 다음 조사 시작 시 기록당 한 번 거리 소문(`거리 소문: 어젯밤 누군가 두드리는 소리가 멈췄다.`)과 의심도 +1이 뒤늦게 돌아온다(`meta.survivorNotes`, 자동 저장). 표시해 둔 사람은 후속 조사에서 다시 나타나 구출할 수 있고, 구출하면 기록이 지워진다. 시작 화면에 구출한 생존자 수·이름과 표시해 둔 자리를 표시(변이체 전투·전용 뉴스는 아직 미구현)
- 실종자 흔적방 + 가족 반환 v1: 드물게 `실종자 흔적`방(`missing-trace`)이 나타난다 — 벽에 붙은 사진·이름표, 그 아래 작은 배낭과 삐져나온 가족 편지, 어두운 형체가 멈춘 자리. `조심히 챙긴다`(빛·멘탈을 조금 쓰고 유품을 드러냄, 실제 줍기는 소음 낮음)/`사진만 확인한다`(이름·날짜만 눈에 담는 단서 한 줄, 비밀은 주지 않음)/`그냥 둔다`(멘탈 소소) 중 고른다. 유품(가족 사진·이름표 목걸이·아이 배낭)은 `ITEM_TABLE` 밖에 두어 단서 총량(6)에 영향을 주지 않는다. 지상에서는 가방에 유품이 있을 때만 `가족에게 반환` 출구가 열려, 유품은 가족에게 사례로 일부만 돌려주고 나머지는 위원회 반납가로 조용히 넘긴다 — 암시장/위원회보다 RP는 낮지만 의심도가 눅고 거리 소문이 누그러진다(비밀은 해금하지 않음). 캐시는 `game.js?v=39`로 범프
- 핵심 행동 사운드 v1: WebAudio로 코드에서 생성하는 짧은 큐 — 오디오 파일도 네트워크 요청도 없다. 챙기기(천/금속 틱)·버리고 도망(긁힘)·판매/가족 반환(낮은 확인 펄스)·강화 성공(오르는 차임)·기절(둔탁한 낙하), 그리고 기척이 near·contact로 **올라갈 때만** 낮은 맥동 하나가 난다(`run.lastPresenceSfx` 엣지 트리거로 매 렌더/틱 반복 없음). 마스터 게인은 0.16으로 낮게 둔다. 소리는 첫 사용자 제스처(`pointerdown`/`touchstart`) 이후에만 시작하고(브라우저 자동재생 정책 준수), `AudioContext`가 없거나 막힌 환경에서는 조용히 무시한다(throw 없음). 시작 화면 하단의 `소리`/`무음` 토글로 끌 수 있고 선택은 저장된다(`localStorage: unlit-halls-sound-off`). 캐시는 `game.js?v=41`·`styles.css?v=25`로 범프
- 문구 정리 v1 (copy audit): 플레이어에게 보이는 대사·버튼 부제에서 흔적·꼬리·기록이 어딘가에 이어진다는 식의 뜻이 흐릿한 상투구를 몸·바닥·기록에 걸리는 구체적 표현으로 바꿨다 — 예) `구조 기록에 이름이 찍혔다`, `손등에 검은 얼룩이 번졌다`, `꼬리를 덜 잡힌다`, `한동안 마음이 무겁다`. 게임 규칙·상수·저장 형식·견적은 그대로다(`보유 RP`·`위로금`처럼 자연스러운 표현은 유지). 캐시는 `game.js?v=43`로 범프(CSS 불변)
- 회수물 태그 `marked` v1 (위원회 표식): 봉인 데이터칩·봉인 유물에 `marked` 태그를 달아 `itemMarked` 헬퍼로 읽는다(태그 누락 시 false). 인던에서는 물건을 집을 때 `위원회 직인이 작게 찍혀 있다`는 짧은 물리 힌트만 흘리고 수치·명단은 감춘다. 판매는 안전 제출 쪽으로 살짝 기운다 — `안전 제출`(위원회)은 표식 물건당 의심도를 조금 더 덜고(그 물건이 공개 수배·검문 목록에서 빠짐), `위험 판매`(암시장)는 표식 물건당 의심도가 조금 더 오른다(표식된 물건이 뒷골목에 다시 떠올라 순찰이 출처를 캔다). 양방향 보정 모두 한 판매에서 상한이 있어 한쪽 경로가 항상 최선이 되지 않는다. 가족 반환은 유품 자체(표식 없음)를 건드리지 않고, 유품이 아닌 표식 짐이 위원회 반납분으로 함께 넘어갈 때만 그만큼 눅는다. 장비 화면 요약에 처리 결과 한 줄이 뜬다. 저장 형식·`SAVE_VERSION`·단서 총량(6)은 불변. 캐시는 `game.js?v=44`로 범프(CSS 불변)
