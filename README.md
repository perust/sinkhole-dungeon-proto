# 도심 싱크홀 — 회수 잠수 프로토타입

가볍고 직관적인 던전 회수(extraction) 게임의 첫 플레이어블 프로토타입입니다.

## 핵심 훅

> 던전이 내가 훔친 물건을 다시 가져가려 한다.

플레이어는 도심 싱크홀 아래 던전에 들어가 회수물을 훔치고, 회수자가 물건을 되찾으러 쫓아오기 전에 지상으로 귀환해 판매합니다. 얻은 RP로 가방/조명/장비 중 하나를 강화하고 다시 더 깊이 내려갑니다.

## 구현된 루프

1. `입장`으로 던전 1층 진입 — 층마다 5~7개 노드짜리 작은 맵이 생성된다
2. 갈림길 선택: 현재 노드의 출구로 동적 구성(예: `왼쪽 문` / `정면 복도` / `환풍구` / `계단 아래로` / 막다른 `관리실`)
3. 도착한 노드에 회수물이 있으면 `줍기`로 가방에 추가
4. 내부 위험도 상승. 화면에는 숫자 대신 `기척`만 남음
5. 몬스터 이벤트 대응: 복도 끝의 소리 / 갈림길을 스치는 발소리 / 움직이면 들킴(`기다리기`로 완화)
6. `계단 아래로` / `버리고 도망` / `들고 귀환` 판단
7. 귀환 성공 시 판매처 선택: `위원회` 또는 `암시장`
8. 판매 결과로 RP, 의심도, 진실 조각, 의뢰 결과가 변화
9. 가방/조명/장비 중 하나 강화
10. `다시 내려가기`로 다음 런 시작
11. 내부 위험도가 한계에 닿으면 회수자가 가방을 되찾아가고 런 실패, 일부 보상만 유지

## v2 고도화 반영

- 판매처 선택: 위원회는 안전/저가/의심도 감소, 암시장은 고가/의심도 상승/진실 조각 해금
- 지상 의심도: 장기 압박 자원
- 진실 조각: 회수물별 미스터리 해금
- 오늘의 의뢰: 반복 진입의 단기 목표
- 다음 목표/거리 뉴스/런 요약: 플레이어가 왜 다시 들어가야 하는지 명확화
- 저장/이어하기: RP, 강화, 최고 깊이, 의심도, 진실 조각, 오늘의 의뢰 진행 자동 저장
- 모바일 앱형 UI: PWA manifest, 앱 셸, 상단 상태바, 하단 조작 독
- 1인칭 던전 탐색: 설명 패널을 줄이고, 갈림길 선택 → 전진 → 발견 흐름으로 변경
- 작은 맵 생성: 층마다 5~7개 노드(출구/방 유형/아이템/몬스터)로 구성된 미로
- 동적 갈림길: 항상 3개 고정이 아니라 현재 노드 출구에 따라 2~4개의 상황형 선택지
- 몬스터 시야 이벤트 3종: 직선 끝에 보임 / 갈림길을 지나감 / 움직이면 들킴 + `기다리기`
- 시야창 확대 + 줍기 전/후 애니메이션 + `앞으로` 전진 시 문 접근 연출 강화
- 비가시 회수자 연출: 회수자 스프라이트/추적도 숫자/위험 바를 숨기고 `기척` 감각 힌트만 표시
- 강화 화면 고정 UI: 판매/의뢰/뉴스가 길어져도 강화 3종과 `다시 내려가기` CTA가 화면 안에 고정
- 공간 배치형 장소 선택: 장소 버튼을 좌/중/우 방향 패드처럼 배치하고, 긴 상황 설명은 가방 아래 cue 박스로 분리
- 아이템 태그 v1: 물건별 `heat`(암시장 의심도)·`noise`(집기 소음 low/medium/high)·`fragile`(버리면 파손). 인던에서는 수치 대신 `소리가 클/깨질 것 같다` 힌트만 노출
- 진실 조각 6개 엔딩 v1: 마지막 조각을 처음 해금하면 강화 화면 대신 엔딩(리빌 3줄 + 6/6)을 띄우고, `심층 루프 계속`(강화 루프 유지)/`기록 초기화`를 제공. `endingSeen` 저장 플래그로 확인 여부 기록

- 끌개 v1 (버린 물건 자국): `버리고 도망`이 물건을 없애지 않고 바닥에 임시 자국으로 남긴다. 던전은 미끼가 아니라 되찾을 물건을 좇아 그 칸으로 향하고, 유예 안에 회수해 간다. 플레이어가 놈보다 먼저 그 칸에 닿으면 다시 챙길 수 있다(`그냥 둔다`를 고르면 자국은 남아 놈의 몫이 된다)
- 방 타입 확장 v1 (무너진 통로 · 위원회 감시소): `무너진 통로`는 고위험/고보상 방으로, `잔해를 넘어간다`(빛·멘탈 소모·소음↑, 방에 물건이 없으면 더 나은 회수물이 드러남)/`낮게 돌아간다`(시간·빛은 들지만 조용)/`포기한다`를 고른다. `위원회 감시소`는 드물게 등장하며 의심도와 상호작용한다 — `기록을 지운다`(의심도 소폭↓·소음)/`단말을 뒤진다`(단서 로그나 물건 기회, 의심도 높으면 위험↑)/`그냥 지나간다`. 인던에서 수치·진실 조각을 공짜로 노출하지 않고 단서 한 줄만 흘린다

상세 설계는 `GAME_SYSTEM_V2.md`, 남은 작업은 `NEXT_STEPS.md`를 참고합니다.


## 실행 방법

빌드 과정이 없는 정적 HTML 프로토타입입니다.

```bash
cd /opt/data/repos/sinkhole-dungeon-proto
python3 -m http.server 8125 --bind 127.0.0.1
```

브라우저에서 열기:

```text
http://127.0.0.1:8125/
```

## 파일 구조

- `index.html` — 화면 구조
- `styles.css` — 모바일 세로 HUD/UI 스타일
- `manifest.webmanifest` — 모바일 홈 화면/PWA 실행 정보
- `src/game.js` — 게임 상태, 회수/추격/귀환/판매/강화 루프
- `assets/pixel/` — 생성형 도트 캐릭터, 몬스터, 아이템, UI, 배경 에셋

## 현재 범위

- 던전 3층
- 회수자 1종
- 회수물 6종
- 강화 3종: 가방 / 조명 / 장비
- 위원회/암시장 판매처 선택
- 의심도와 진실 조각 메타 진행
- localStorage 기반 저장/이어하기와 기록 초기화
- 층별 작은 맵 생성과 동적 갈림길 선택
- 모바일 앱형 상단 HUD + 하단 조작 독
- 1인칭 복도 시야, 장소 선택, 발견 후 행동 버튼
- 몬스터 기척 이벤트와 상황형 `기다리기` 선택
- 추적 수치 없는 공포 연출: 내부 위험도는 유지하되 UI는 `정적/먼곳에서/잡음/금속음/숨소리`만 표시
- 공간형 하단 선택지: 왼쪽/정면/오른쪽/낮은 길/뒤로 가기가 실제 위치감 있게 배치되고, 설명은 버튼 밖 cue에 표시됨
- 생성형 도트 에셋을 적용한 시작 화면, 던전 배경, 캐릭터, 아이템 UI
