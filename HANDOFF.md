## 완료된 패스

### Pass 1 — 저장/이어하기

구현됨:

- `localStorage` 저장 키: `sinkhole-dungeon-save`
- 저장 버전: `version: 1`
- 저장 대상: RP, 강화 레벨, 최고 깊이, 총 획득 RP, 의심도, 진실 조각, 오늘의 의뢰 진행
- 깨진 JSON/버전 불일치/범위 밖 값 안전 처리
- 시작 화면 `기록 초기화` 버튼
- 자동 저장 지점: 최고 깊이 갱신, 판매처 선택, 의뢰 완료, 실패 보상, 강화 구매

검증됨:

- 저장값 생성
- 새로고침 후 복원
- 기록 초기화 후 localStorage 제거 및 시작 화면 초기화

### Pass 2 — 방 선택 3지선다 / 방 타입 3개

구현됨:

- 층 진입 시 바로 회수물이 뜨지 않고 방 선택을 먼저 보여줌
- 방 타입 3종:
  - 안전 캠프: 조명 회복, 낮은 위험
  - 폐쇄 연구실: 희귀/진실 단서 확률 상승, 약간의 비용
  - 회수자 둥지: 고가 회수물 확률 상승, 추적도 압박 증가
- 방 선택 전에는 기존 액션 버튼을 숨겨 모바일 화면에서 선택지가 잘리지 않게 함
- 방 선택 후 기존 루프(`챙기기 → 탈출 → 판매처 선택`) 유지

검증됨:

- `node --check src/game.js`
- `git diff --check`
- 브라우저 로컬 검증:
  - 내려가기 후 방 선택 3개 표시
  - 방 선택 후 회수물 표시
  - 챙기기 후 판매처 선택 화면 진입
  - 방 선택 중 모바일 화면에서 버튼 잘림 없음

### TRPG Pass — 방 선택 판정

완료됨:

- 방 선택마다 `d20 + 보정치 ≥ 난이도` 판정 추가
- 보정치 요소:
  - 조명이 충분하면 유리
  - 조명이 낮으면 불리
  - 빈 가방은 유리, 거의 찬 가방은 불리
  - 안전 캠프는 유리, 회수자 둥지는 불리
- 결과 4종:

  - 실패: 조명 손실, 위험 상승, 추격 시작 가능
- 모바일 첫 viewport 안에 d20 판정 카드 표시
- 기존 핵심 루프는 유지: 방 선택 → 회수물 확인 → 챙기기 → 탈출 → 판매처 선택

검증됨:

- `node --check src/game.js`

- `git diff --check`

  - 방 선택 후 `d20`, 결과, 보정치, 난이도 표시

### Exploration Pass — 작은 맵 + 동적 선택지 + 비가시 몬스터 이벤트

완료됨:

- `run.floorMap` / `run.currentNodeId` / `run.holdEvent` 최소 맵 상태 추가
- 각 층 진입 시 5~7개 노드짜리 작은 맵 생성(`generateFloorMap`)
  - 무방향 연결 그래프(등뼈 + 가지 + 여분 연결), 입구 차수 ≥ 2
  - 계단(다음 층 입구)은 트리상 가장 깊은 잎으로 두어 항상 직전 노드를 지나가게 함
  - 노드마다 `exits` / `kind` / `item?` / `monster?`
- 선택지는 `node.exits` 기반으로 동적 렌더링(2~4개). 3개 고정 아님.
  - 상황형 label: 왼쪽 문 / 정면 복도 / 낮은 창고 / 관리실 / 환풍구 / 무너진 통로 / 오른쪽 균열 / 계단 아래로
  - 막다른 노드는 `(막다른)` 표기
- 아이템은 노드에 배치(입구/계단 제외, 2~3개). 도착한 노드에서만 발견물 표시
- 몬스터 이벤트 3종 모두 구현하되, 회수자 접근/추적 수치는 직접 보여주지 않음:
  - `sight`: 복도 끝의 젖은 쇳소리 — 내부 위험 상승/로그, 그쪽 출구는 `젖은 쇳소리`로 표시, 전진 시 위험 추가
  - `cross`: 갈림길을 스치는 발소리 — `기다리기`로 흘려보내거나 다른 길; 지나가는 중 이동하면 들켜 추격 시작
  - `ambush`: 옆에서 멈춘 무언가 — 다음 이동 시 위험 급증·추격; `기다리기`로 완화
  - `기다리기`는 `holdEvent`가 있을 때만 동적으로 노출
- 1층은 계단 직전 노드에 `sight`를 보장 배치(첫 인카운터 보장, 부드러움 유지)
- UI: 중앙 stage 확대, 줍기 버튼을 시야 안으로, 줍기 전/후 애니메이션, `앞으로` 전진 시 중앙 문 접근 연출 강화, 단일 하단 dock(동적 선택지 + 보조 행동)
- 강화 화면 고정 UI: 판매/의뢰/뉴스가 길어져도 강화 3종과 `다시 내려가기` CTA가 화면 안에 유지
- 비가시 회수자 연출: 회수자 스프라이트, 추적도 숫자, 위험 바를 숨기고 `기척` 감각 힌트만 표시
- 방향키형 장소 선택: 출구 버튼에 `←/↑/→/↙/↩` glyph와 방향 class를 붙여 방향 이동처럼 보이게 함
- 기존 판매처/저장/강화/실패/의뢰 루프는 그대로 유지

제외(요청대로 미구현):

- 진실 조각 엔딩, 아이템 heat 태그 개별화, 새 에셋, dependency 추가

검증됨:

- `node --check src/game.js`
- `python3 -m json.tool manifest.webmanifest`
- `git diff --check`
- 브라우저 로컬 검증:
  - 동적 선택지 2~4개, 아이템 노드, 줍기→판매→강화 루프
  - 강화 화면에서 577px 높이 기준 강화 3종과 `다시 내려가기` 모두 표시
  - 던전 화면에서 `추적도` 텍스트/숫자/위험 바/회수자 스프라이트 미노출, `기척`만 표시
  - 방향 glyph가 크게 보이고 장소 버튼이 잘리지 않는지 브라우저 비전 확인
- 헤드리스 맵 불변식 하니스(9,000개 맵): 연결성, 계단 도달성/잎 보장, 1층 인카운터 보장, 아이템/몬스터 배치 규칙 (697,726 assertions 통과)
- 브라우저 로컬 검증 방법은 README의 `python3 -m http.server`로 가능

### Pass 3 — 아이템 Heat/태그 개별화

권장 범위:

- 현재 등급 기반 Heat를 아이템 개별 속성으로 분리한다.
- 우선 MVP 속성은 `heat`, `noise`, `fragile` 세 개만 넣는다.
- `heat`: 암시장 판매 시 의심도 상승량
- `noise`: 챙기는 순간 추적도 상승량
- `fragile`: 버리고 도망 시 보상 일부만 남는 물건
- 판매처/의심도/챙기기 위험/버리기 계산에 새 속성을 연결한다.

제외:

- 진실 조각 6개 엔딩
- 새 회수자
- 출구 선택
- 그래픽 고도화
