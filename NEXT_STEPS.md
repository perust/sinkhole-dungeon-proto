# Sinkhole Dungeon Proto — 다음 작업 정리

현재 프로토타입은 `챙기기 → 도망/귀환 → 판매처 선택 → 강화 → 재진입` 루프와 도트 에셋 적용까지 완료된 상태입니다. 아래는 다음 단계에서 작업하면 좋은 항목입니다.

## 1. 저장/이어하기

- `localStorage`로 RP, 강화 레벨, 최고 깊이, 의심도, 진실 조각 저장
- 시작 화면에 `새로 시작` / `이어하기` 또는 `기록 초기화` 선택 추가
- 저장 데이터 버전 필드 추가해서 이후 밸런스 변경 시 마이그레이션 가능하게 만들기

## 2. 진실 조각 수집 엔딩

- 진실 조각 6개를 모두 모으면 별도 엔딩/결말 화면 표시
- 엔딩 후보:
  - 위원회가 싱크홀을 관리하는 게 아니라 키우고 있었다
  - 회수자는 괴물이 아니라 위원회 방범 시스템이었다
  - 플레이어가 들고 나오는 유물이 싱크홀을 더 깊게 여는 열쇠였다
- 엔딩 이후 `더 깊은 회차` 또는 `기록 초기화` 선택 제공

## 3. 방/층 다양화

- 현재는 층 3개와 회수 포인트 중심 구조
- 다음 방 타입 후보:
  - 안전 캠프: 조명 일부 회복
  - 무너진 통로: 조명 소모 증가, 더 깊이 보상 증가
  - 폐쇄 연구실: 희귀 회수물 등장률 증가
  - 회수자 둥지: 위험 상승이 빠르지만 고가 아이템 등장
  - 위원회 감시소: 의심도가 높을수록 불리한 이벤트

## 4. 몬스터/회수자 연출 강화

- 추적도 50/80/95% 구간별 시각 연출 추가
- 회수자 등장/접근 애니메이션 강화
- 위험도 높을 때 버튼 흔들림, 화면 암전, 경고 문구 추가
- 실패 시 잃은 아이템을 더 명확히 보여주기

## 5. 아이템 차별화

- 현재 회수물은 `칸 수 / 가치 / 등급 / 진실 조각` 중심
- 추가하면 좋은 속성:
  - `heat`: 암시장 판매 시 의심도 증가량
  - `noise`: 챙기는 순간 추적도 증가량
  - `fragile`: 버리고 도망 시 깨져서 보상 일부 획득
  - `marked`: 위원회 판매 시 의심도 더 많이 감소
- 아이템별 도트 아이콘을 개별 스프라이트로 잘라 더 명확히 표시

## 6. 모바일 UX polish

- 짧은 화면에서 모든 화면이 한 화면 안에 들어오는지 계속 확인
- 버튼 터치 영역과 간격 조정
- 시작 화면의 아이템 프리뷰가 너무 작으면 단순화
- HUD가 조금 붐비므로 `RP / 최고 / 가방`을 더 간결하게 정리

## 7. 사운드/피드백

- 챙기기, 위험 상승, 판매, 강화, 실패 사운드 추가
- 조명 낮음/추적도 높음 경고 효과음
- 강화 성공 시 짧은 반짝임 또는 진동 느낌의 CSS 애니메이션

## 8. 배포/품질 체크

- GitHub Pages 반영 후 라이브 URL에서 직접 플레이 확인
- 브라우저 콘솔 에러 확인
- `node --check src/game.js`
- `git diff --check`
- 모바일 폭 390px~430px 기준 주요 화면 확인

## 우선순위 추천

2. 진실 조각 6개 엔딩
3. 방 타입 3~5종
4. 아이템별 위험/보상 차별화
5. 사운드와 애니메이션 polish
