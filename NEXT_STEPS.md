### 1. 아이템 Heat/태그 고도화

현재는 등급 기반 Heat에 가깝다. 다음에는 아이템 개별 속성으로 분리해야 함. 방 선택 3지선다의 전략성을 살리려면 엔딩보다 먼저 이 작업이 필요하다.

**v1 완료 (heat/noise/fragile):** 6종 물건에 `heat`·`noise`·`fragile` 태그 추가, `itemHeat`/`itemNoise`/`itemFragile` 헬퍼로 접근하며 태그 누락 시 등급 기준으로 보정.

**`family` 태그/루트 v1 추가:** `itemFamily` 헬퍼와 실종자 흔적방 유품(`FAMILY_KEEPSAKES`)·지상 가족 반환 경로(아래 방 타입 확장 참고).

**`marked` 태그 v1 추가:** `itemMarked` 헬퍼(태그 누락 시 false)와 봉인 데이터칩·봉인 유물에 `marked: true`. 인던 물건 힌트에 `위원회 직인이 작게 찍혀 있다` 한 줄, 판매 견적에 결정적·유계 보정(안전 제출↓·위험 판매↑)과 강화 화면 요약 한 줄.

**`truth` 태그 v1 추가:** `itemTruth`/`itemTruthText` 헬퍼로 "단서 여부(`truth: true`)"와 "해금 문구(`truthText`)"를 분리했다. `TRUTH_TOTAL`은 `itemTruth`로만 세어 앞으로 truth 없는 일반 회수물이 섞여도 6/6 목표가 흔들리지 않고, 구버전 문자열 `truth`도 하위호환으로 참·문구 처리한다. 저장 형식·`SAVE_VERSION` 불변.

- [x] `heat`: 암시장 판매 시 의심도 상승량 (물건별 합산: 배터리 3·배관 2·데이터칩 9·노트 7·코어 15·유물 20)
- [x] `noise`: 집는 순간 추적 압박 (low/medium/high, 조심/재빨리와 조합)
- [x] `marked`: 위원회가 추적 중인 회수물 — 봉인 데이터칩·봉인 유물에 `marked: true`, `itemMarked` 헬퍼(태그 누락 시 false). 인던에서 `위원회 직인이 작게 찍혀 있다` 물리 힌트만 노출(수치·명단 비노출). 판매를 안전 제출 쪽으로 살짝 기울인다 — `안전 제출`(위원회)은 표식 물건당 의심도 -2(추적 명단 소거), `위험 판매`(암시장)는 표식 물건당 의심도 +3(중개상 재확인). 양방향 모두 판매당 상한 6으로 묶어 한쪽이 항상 최선이 되지 않게 하고, 견적·의심도에 결정적으로 반영돼 귀환 화면 미리보기에 그대로 보인다. 가족 반환은 유품(표식 없음)을 건드리지 않고 유품이 아닌 표식 짐이 위원회 반납분으로 넘어갈 때만 눅는다. 강화 화면 요약에 처리 결과 한 줄(`markedSaleNote`). `SAVE_VERSION`·저장 형식·단서 총량(6) 불변. `game.js?v=43`→`v=44` 범프
- [x] `truth`: 단서 여부(태그) — 물건에 `truth: true` + `truthText`로 분리, `itemTruth`/`itemTruthText` 헬퍼(구버전 문자열 `truth`는 참·문구로 하위호환). `TRUTH_TOTAL`은 `itemTruth`로만 세어 앞으로 truth 없는 일반 회수물이 섞여도 6/6 목표 불변. 암시장 해금은 `itemTruth`일 때만·문구는 `itemTruthText`로 읽는다. 위원회·가족 경로는 비밀을 풀지 않는다. 저장 형식·`SAVE_VERSION` 불변. `game.js?v=44`→`v=45` 범프
- [x] `family`: 실종자 흔적방 유품(`itemFamily`, `FAMILY_KEEPSAKES`) — `ITEM_TABLE` 밖에 둬 `TRUTH_TOTAL`=6 유지, 지상에서 `가족에게 반환` 경로로 연결(아래 방 타입 확장 v1 참고)
- [x] `fragile`: 버리고 도망 시 파손 — 결과 문구에 `깨지는 소리가 났다` 노출 (데이터칩·노트·유물)

### 3. 방 타입 확장 — v1 완료 (무너진 통로 · 위원회 감시소 · 실종자 흔적방)

MVP 3종에 이어 방 타입 확장 v1로 세 종(무너진 통로·위원회 감시소·실종자 흔적방)을 추가했다.

**v1 완료:** `무너진 통로`를 일반 발소리 방에서 분리해 고위험/고보상 방으로 만들고, `위원회 감시소`(`watchpost`)와 `실종자 흔적방`(`missing-trace`)을 드물게 등장하는 새 방 타입으로 추가했다. 인던에서 수치·비밀을 공짜로 노출하지 않고 단서 로그 한 줄만 흘린다.

- [x] 무너진 통로: `collapsed-passage` 이벤트. `잔해를 넘어간다`(빛·멘탈 소모·소음↑, 방에 물건 없으면 `pickFloorItem(style:danger)`로 더 나은 회수물을 드러냄 — 이미 있으면 중복 생성 안 함)/`낮게 돌아간다`(빛은 들지만 소음·위험↓)/`포기한다`. 잔해·철근·먼지·들보 삐걱임을 구체적으로 묘사
- [x] 위원회 감시소: `watchpost` 이벤트, `meta.suspicion`과 상호작용. `기록을 지운다`(의심도 -3·빛/멘탈 소모·작은 소음)/`단말을 뒤진다`(의심도 낮으면 단서 로그나 물건 기회, `WATCHPOST_TENSE_SUSPICION`(40) 이상이면 경보로 위험·의심도↑)/`그냥 지나간다`. `WATCHPOST_SPAWN_CHANCE`(0.28)로 드물게, 입구/계단 제외 배치
- [x] 실종자 흔적방: `missing-trace` 방(uncommon, `MISSING_TRACE_SPAWN_CHANCE` 0.24, 층당 최대 하나·입구/계단/감시소 노드 제외). 벽 사진/이름표/배낭/가족 편지에서 `조심히 챙긴다`(빛·멘탈 조금 쓰고 유품을 드러냄, 실제 줍기는 소음 low)/`사진만 확인한다`(이름·날짜 단서 한 줄, 비밀은 주지 않음)/`그냥 둔다`(멘탈 소소)를 고른다. 유품(`FAMILY_KEEPSAKES`, `family:true`)은 `ITEM_TABLE` 밖이라 `TRUTH_TOTAL`=6 유지
- [x] 가족 반환 v1: 가방에 유품(`family`)이 있을 때만 귀환 화면에 `가족에게 반환`(`buy-family`) 버튼이 뜬다(없으면 숨겨 선택지 잡동사니를 줄임). 유품은 가족에게 사례(`FAMILY_RETURN_RATE` 0.4)로만 돌려주고 나머지는 위원회 반납가(0.72)로 조용히 넘긴다 — 암시장/위원회보다 RP는 낮지만 의심도가 눅고(`FAMILY_RETURN_SUSP_RELIEF` 2×유품 수 + 반납 물건 수 완화) 거리 소문이 누그러진다. 비밀은 해금하지 않는다
